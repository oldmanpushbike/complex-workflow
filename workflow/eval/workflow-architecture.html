01 / ARCHITECTURE

三层架构总览

每一层都建立在下一层之上。L2 的确定性脚本强制执行 L1 定义的流程，L3 度量一切并驱动持续改进。

🔄

L1 · 主循环层

功能开发主干 + 反馈回退 + 跨功能学习弧。失败不是死胡同——是有方向的信号。

  • S0 → S9 完整状态机
  • 门禁失败 → 反馈注入 → 回退重试
  • 48h/72h 停滞检测 + 人类升级
  • 单Agent↔双Agent 模式切换
  • Retro → OpenSpec 跨功能学习弧
  • feedback-loop.md (844行) + resume-protocol.md (793行)
🔒

L2 · 确定性门禁层

门禁不是"建议"——是硬阻断。dispatcher 将主会话退化为纯执行器，不再依赖 Agent 的自觉。

  • dispatch.sh — 硬编码路由决策树，零LLM
  • gate-check.sh — 7门禁产物检查 + --enforce 硬阻断
  • validate-state.sh — JSON Schema 校验 + --fix 自修复
  • ci-gate-guard.sh — CI/CD 门禁守护
  • .claude/settings.json — PreToolUse/PostToolUse Hook 实时拦截
  • feature-state.schema.json — 结构化状态唯一真相源
📊

L3 · 评估与进化层

度量一切，沉淀一切。评分引擎告诉人类工作流在变好还是变差，经验管道确保每个坑只踩一次。

  • score.py — 7维评分引擎（纯算术，零LLM）
  • dashboard.html — 可视化仪表盘
  • extract-lessons.sh — Lesson 自动提取
  • pipeline.md — Lesson→Pattern→Instinct 三级进化
  • EPR (经验沉淀率) 指标追踪
  • baseline 管理 + 确定性 Hash 校验

06 / EXPERIENCE PIPELINE

经验三级进化管道

Lesson（教训）→ Pattern（模式）→ Instinct（本能规则）。每级晋升需人工确认——防止错误经验扩散。这是项目产生"复利"的核心机制。

📝

Level 1 · Lesson

来源：07-task-retro.md 自动提取
触发：每次 retro 完成
存储：lessons/<feature-id>/lesson-NNN.md
状态：draft → reviewed → promoted
确认：无需确认（自动提取）
范围：单功能/单任务
→
🧩

Level 2 · Pattern

由 ≥2 个不同功能的 Lesson 归纳
晋升条件：≥2个不同功能 + ≥1个P0/P1
存储：patterns/<pattern-name>.md
确认：人工（六段Checkpoint格式）
内容：症状/根因/触发条件/解决方案/Instinct候选
→
🧬

Level 3 · Instinct

Pattern 激活 ≥30天 + recurrenceCount ≥3
晋升条件：≥30天 + ≥3次复发 + 可机械执行
注入：AGENTS.md / CLAUDE.md / state-machine.md
确认：人工签核（人类执行写入）
逆转：每条Instinct必须定义逆转条件
📈 EPR (经验沉淀率) = (活跃Pattern数 + 活跃Instinct数) / (总Lesson数 + 1) × 100。 ≥30% 优秀 · 15-29% 健康 · 5-14% 警告 · <5% 危险。 详细定义 →

08 / QUICKSTART

快速上手：五步启动一个功能

从零到完成的日常实践流程。

读取上下文

读 AGENTS.md 了解共享契约。确定自己是编排者还是挑战者。

运行 Dispatcher

bash workflow/scripts/dispatch.sh <feature-id> --json
获取机器可读的路由指令。

照指令执行

严格按 nextAction 中的 action 和 requiredReads 执行当前状态的任务。

门禁检查

状态转换前运行 gate-check.sh。若失败 → feedback-loop：注入修正 → 回退 → 重试。

沉淀经验

S8 完成 retro → 运行 extract-lessons.sh。Pattern 候选满足条件时人工确认晋升。

09 / SCORING

七维评分引擎

score.py 实现。每个维度 0-100 分，纯算术，零 LLM。SHA256 hash 确保不可篡改。

维度权重度量内容公式方法
D1 流程完整性18%所有门禁产物是否存在且状态为passedgate artifacts 文件存在性 + 大小阈值 + 状态检查
D2 产物质量15%门禁检查结果的完备性，产物是否通过内容校验checkResults.allArtifactsPresent 覆盖率 + checkResults 字段丰度
D3 代码正确性18%审查发现中 P0/P1 密度、修复率、单Agent折扣density(50) + severity(50) + fixBonus(已修复×5)，single-agent×0.7
D4 效率12%状态流转耗时 vs 预期基准转移耗时阶梯函数：≤预期=100，超出按10分/h衰减
D5 安全合规15%安全类发现数量、P0暴露风险、修复完成率P0×30 penalty + 未修复×12 penalty，无安全发现=100
D6 迭代能力12%反馈循环修正成功率与修复效率retryHistory 成功率×20 + feedbackInjected + 未达上限bonus
D7 接口验收10%验证门禁状态、验收标准通过、人类签核Gate6状态阶梯 + 验证发现+验收决策bonus
85-100 A 优秀 70-84 B 良好 55-69 C 及格 40-54 D 需改进 0-39 F 不及格
